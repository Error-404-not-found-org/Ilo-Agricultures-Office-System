HOW TO BUILD A STANDALONE ANDROID APK (NO PLAY STORE)
=========================================================
Since this app is built with Expo, you will use EAS (Expo Application Services) to generate an APK that you can directly install on any Android phone (sideloading) without going through the Google Play Store.

STEP 1: Install EAS CLI
Open your terminal and run:
`npm install -g eas-cli`

STEP 2: Log into your Expo Account
Run:
`eas login`
(Enter your Expo account credentials)

STEP 3: Configure the Project for EAS
Navigate into your `mobile` folder in the terminal and run:
`eas build:configure`
This will generate an `eas.json` file in your mobile directory.

STEP 4: Modify eas.json to Output an APK
By default, Expo builds an `.aab` file for the Play Store. To force it to build an `.apk` file instead, open `eas.json` and modify the "preview" profile like this:
{
  "build": {
    "preview": {
      "android": {
        "buildType": "apk"
      }
    },
    "production": {}
  }
}

STEP 5: Start the Build Process
Run this command to send the project to Expo's cloud servers to build the APK:
`eas build -p android --profile preview`

STEP 6: Download & Install
Once the build is complete (it usually takes 5-10 minutes), the terminal will give you a direct download link. You can also log into your Expo dashboard on your browser to download the `.apk` file. 
Transfer that `.apk` file to your Android phone, tap it, and select "Install" to test the fully compiled production app!
